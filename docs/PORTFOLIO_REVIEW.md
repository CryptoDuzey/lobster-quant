# 龙虾量化作品集审视报告

这份文档用于说明当前项目作为作品集展示时，哪些能力已经可以真实演示，哪些边界需要诚实说明，以及后续最值得继续打磨的方向。

## 一句话定位

龙虾量化是一个面向中国 A 股市场的 AI 原生量化投研工作站。它把股票行情、自然语言策略生成、真实回测、策略保存、AI 分析和数据缓存整合到一个可运行的 Web 产品里。

## 当前已经适合作品集展示的能力

1. 行情雷达
   - 支持 A 股股票搜索。
   - 支持真实日线 K 线展示。
   - 支持实时行情快照。
   - 支持消息面展示和 AI 行情助手。

2. 自然语言策略工坊
   - 支持普通对话。
   - 支持把自然语言策略转成可回测规则。
   - 支持多类规则策略：均线、RSI、布林带、MACD、突破、成交量过滤等。
   - 支持基础机器学习择时实验，但明确不冒充 XGBoost 或深度学习。

3. 可信回测实验室
   - 回测结果来自真实行情数据。
   - 不同策略会产生不同的策略 Hash、代码 Hash、交易次数和收益曲线。
   - 回测报告展示累计收益、基准收益、Alpha、Beta、Sharpe、最大回撤、年化收益。
   - 基准曲线无效时不会画假线。
   - 分钟数据不足时会明确失败，不用日线或假数据冒充。

4. 数据持久化
   - SQLite 保存回测记录、策略、行情缓存、基准缓存、模型配置、API Key 配置和 Agent/Skill 配置。
   - 本地重启后不会像新项目一样丢失历史记录。
   - AI / Agent 调用审计日志已写入数据库，同时保留本地 JSONL 文件。

5. 部署与工程
   - 前端支持 Vite 构建。
   - 后端支持 FastAPI 启动。
   - 支持 Docker 部署到魔搭社区。
   - 提供本地启动、停止和产品烟测脚本。

## 需要诚实说明的边界

1. 分钟级长历史
   - 当前会尝试东方财富、新浪和腾讯等真实分钟接口。
   - 如果数据源无法覆盖长期分钟历史，系统会报错，不会伪造。
   - 未来要做稳定分钟级长期回测，建议接入更稳定的数据服务或自建分钟数据仓库。

2. XGBoost / 深度学习
   - 当前只开放基础机器学习实验。
   - 不会把基础线性模型包装成 XGBoost、LSTM 或 Transformer。
   - 若要正式支持，需要补齐训练集管理、特征版本、样本外验证、模型保存和回测接入。

3. 多因子股票池
   - 当前已有基础股票池 / 因子实验链路。
   - 但完整生产级多因子仍需要更完善的成分股历史、财务因子、行业中性化和组合调仓验证。

4. 实盘交易
   - 当前不开放实盘下单。
   - 项目定位是投研、策略生成、回测和纸面验证。

## 今天之后最值得继续打磨的方向

1. 数据层
   - 增强分钟级数据源。
   - 增加云端持久化数据库。
   - 做数据质量评分：覆盖率、缺失率、截断提示、来源标记。

2. 策略泛化
   - 扩大自然语言规则解析能力。
   - 增加策略解释器，支持更多表达式组合。
   - 对无法回测的策略先转成研究方案，而不是硬跑。

3. UI 作品集观感
   - 首页保持 K 线为中心，减少左右面板压迫。
   - 工坊继续向 ChatGPT / BigQuant 式对话体验靠拢。
   - 回测页突出六个核心指标、收益曲线和交易流水。
   - 广场页加强策略详情、评论、收藏和作者信息。

4. 作品集叙事
   - 突出“我不是做了一个静态金融页面，而是做了一个真实数据驱动的 AI 量化工作站”。
   - 强调可信回测、防假数据、数据持久化、Agent 分层和部署能力。
